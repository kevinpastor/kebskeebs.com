# kebskeebs.com — Agent Guide

## Stack
- **Framework**: React 19 + TypeScript ~6.0
- **Build**: Vite 8 via TanStack Start (SSR)
- **Routing**: TanStack React Router v1 (`@tanstack/react-router`)
- **CSS**: Tailwind CSS 4 (`@tailwindcss/vite` plugin) + shadcn/ui base-luma style
- **Content**: MDX docs via Content Collections (`@content-collections/*`)
- **UI primitives**: `@base-ui/react`
- **React Compiler**: enabled via `@rolldown/plugin-babel` + `reactCompilerPreset()`

## Commands
| Command | Action |
|---|---|
| `pnpm start` | Dev server |
| `pnpm build` | `tsc -b && vite build` (typecheck first, then build) |
| `pnpm lint` | ESLint |
| `pnpm preview` | Preview production build |

## Architecture
- **Entry point**: Vite resolves `index.html` → SSR entry generated by TanStack Start. No `src/main.tsx` — it is virtual.
- **Routes**: `src/routes/__root.tsx` (layout), `src/routes/index.tsx` (home), `src/routes/docs/$.tsx` (catch-all docs).
- **Content**: `docs/*.md` → compiled by Content Collections to `.content-collections/generated/` (gitignored). The `docs/$.tsx` route renders them via `<MDXContent>`.
- **CSS**: Imported via `?url` in `__root.tsx` and linked as a stylesheet in `<head>`.

## Code conventions & quirks
- **`routeTree.gen.ts` is auto-generated** by the TanStack Router Vite plugin. Do not edit manually.
- **TypeScript ~6.0**: `erasableSyntaxOnly: true` (no enums, no namespaces), `verbatimModuleSyntax: true` (must use `import type` for type-only imports).
- **shadcn aliases** (from `components.json`): `src/components`, `src/lib`, `src/components/ui`, `src/hooks`.
- **CSS variable theme** lives in `src/routes/-index.css` — dark mode via `.dark` class on `<html>`.
- **Content Collections** path alias: `"content-collections"` → `"./.content-collections/generated"` in `tsconfig.app.json`. Import generated content as `import { allDocs } from "content-collections"`.
